Requirements

  • 2 Data Centers per DX location (SJC+Denver, Chi+Dal, ATL+Ash)
  • 3 DX locations: West, Central, East
  • 3+ AWS Regions: us-west-1, us-east-1, eu-central-1 (must scale beyond)
  • All DCs advertise 10.0.0.0/8 summary via BGP
  • All traffic inspected by Network Firewall (single inspection VPC per region)
  • Deterministic routing: each region prefers its local DX
  • Multi-tier failover with traffic engineering control

Recommendation

рҹҸҶ For Scale + Control: Model 3 (Per-Circuit DXGW)

Choose this if you need Cloud WAN-side visibility and deterministic traffic engineering per DX circuit. Scales infinitely. This is what we tested and proved works across all 3 regions with full failover validation.

рҹҘҲ For Simplicity + Scale: Model 2 (Regional DXGW)

Choose this if you have reliable on-prem WAN between DCs and want the simplest possible design. Fewest moving parts. On-prem BGP handles everything within a region.

Choose this if your on-prem WAN is unreliable or doesn't exist between all DCs. The global DXGW ensures AWS backbone carries all failover traffic.

Only if you're certain you'll never exceed 4 regions. The 4 VIF-per-DX-connection limit is a hard wall that requires full redesign to overcome.
